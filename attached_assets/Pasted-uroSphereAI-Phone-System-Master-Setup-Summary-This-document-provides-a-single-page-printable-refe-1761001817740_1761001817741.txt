uroSphereAI Phone System — Master Setup Summary
This document provides a single-page printable reference for the entire AI phone system setup: scripts, files, secrets, non-secrets, connections, ports, endpoints, and URLs.

1. Core Components
AI-Memory Service
File: /root/ai-memory/main.py


Port: 8100


Service Name: ai-memory.service


Purpose: Stores and retrieves conversational memory.


Control Commands:

 systemctl status ai-memory
systemctl restart ai-memory
systemctl stop ai-memory


Endpoints (on https://voice.theinsurancedoctors.com:8100):


/store_memory


/get_memory


/read_memory


/v1/chat/completions


VoiceBridge
File: /opt/voice-bridge/app.py


Port: 9100


Service Name: voice-bridge.service


Purpose: Twilio ↔ ElevenLabs audio bridge.


Control Commands:

 systemctl status voice-bridge
systemctl restart voice-bridge
systemctl stop voice-bridge


SendText Script
File: send_text.py


Port: None (standalone script)


tmux Session: sendtext


Purpose: Outbound SMS notifications.


Control Commands:

 tmux attach -t sendtext
CTRL+b d   # detach
tmux kill-session -t sendtext

6. Workflow Overview
Inbound Call → Twilio forwards audio → VoiceBridge.


VoiceBridge ↔ ElevenLabs for speech synthesis/recognition.


LLM Request sent to RunPod endpoint (Mistral/Falcon/Qwen).


Memory Retrieval from AI-Memory (port 8100).


Response returned → synthesized by ElevenLabs → played back.


Logs & Memory stored in PostgreSQL + AI-Memory.
